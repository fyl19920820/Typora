# 数据结构绪论

数据结构其实就是研究数据对象之前的关系和操作等相关问题的

* 数据：描述信息的元素
* 数据元素：一个完整的数据个体，组成数据的基本单位，对应数据库中的一行记录
* 数据项：数据元素中的每个属性
* 数据对象：数据元素的集合

逻辑结构：就是抽象出来的数据元素之间的关系，像集合，线性，树，图

物理结构：就是实际计算机的存储方式，像数组，队列

抽象是抽取出事物具有的普遍性的本质

抽象数据类型就是把具体的事物进行分解，分解一个个小的数据类型，对其属性及行为进行定义，其实作为一个个独立单元，使问题更好解决

# 算法

数据结构和算法关系：相辅相成

通过两种方式来计算1-100的和，来体现不同算法的之前的优劣

定义：解决特定问题求解步骤的描述，在计算机中表现为指令的有序序列

特性：输入输出，有穷性（不会无限循环下来），确定性（相同的输入只会有唯一的结果），可行性（算法的每一步都是可执行的）

算法设计的要求：

* 正确性：没有语法错误，对于合法的输入，有满足要求的输出，对于不合法的输入，有满足规则说明的输出，对于刁难的测试数据也有满足要求的输出（四层要求，难度递增，最有一层很难满足，一般满足第三层即可）
* 可读性：便于阅读理解
* 健壮性：当输入的数据不合法时，也能作出相关处理，而不是报错或者抛异常
* 时间效率高和存储量低：执行效率高，占用内存空间少

算法效率的度量方法：

* 事后统计方法：通过设计好的程序，通过计算机获取执行时间，从而确定效率高低，  缺点：必须根据算法编好程序，执行时间依赖于计算机的硬件和软件的环境因素，测试数据设计困难，用什么样的测试数据很难判断
* 事前分析估算方法：依据统计方法对算法进行估算
